## Manual Action Guide — Step by Step

### TASK A: Regenerate Password Hashes to PBKDF2 Format

**Why:** Your current `.env` uses raw SHA-256 hashes (no salt). The code now supports PBKDF2 with 310,000 iterations + random salt, but you need to generate new hashes and update `.env`.

#### Step 1: Open your app in the browser
```bash
npm run dev
```
Then open `http://localhost:5173` in Chrome or Edge.

#### Step 2: Open DevTools Console
Press `F12` → Click the **Console** tab.

#### Step 3: Run the hash generation script
Copy and paste this entire block into the console, then press Enter:

```javascript
(async () => {
  const hashPassword = async (plain) => {
    const salt = crypto.getRandomValues(new Uint8Array(16));
    const keyMaterial = await crypto.subtle.importKey(
      'raw',
      new TextEncoder().encode(plain),
      'PBKDF2',
      false,
      ['deriveBits'],
    );
    const bits = await crypto.subtle.deriveBits(
      { name: 'PBKDF2', salt, iterations: 310_000, hash: 'SHA-256' },
      keyMaterial,
      256,
    );
    const toHex = (arr) =>
      Array.from(arr)
        .map((b) => b.toString(16).padStart(2, '0'))
        .join('');
    return `pbkdf2:${toHex(salt)}:${toHex(new Uint8Array(bits))}`;
  };

  const passwords = {
    'ADM': 'YOUR_ADM_PASSWORD_HERE',
    'Gabriela Ferreira': 'YOUR_GABRIELA_PASSWORD_HERE',
    'Sabrina Almeida': 'YOUR_SABRINA_PASSWORD_HERE',
    'Manager': 'YOUR_MANAGER_PASSWORD_HERE',
  };

  console.log('=== GENERATING PBKDF2 HASHES ===\n');

  for (const [user, pwd] of Object.entries(passwords)) {
    const hash = await hashPassword(pwd);
    console.log(`${user}: ${hash}`);
  }

  console.log('\n=== DONE ===');
  console.log('Copy the hashes above and paste them into your .env file.');
})();
```

#### Step 4: Replace the placeholder passwords
Before running, replace these in the script:
- `YOUR_ADM_PASSWORD_HERE` → your actual admin password
- `YOUR_GABRIELA_PASSWORD_HERE` → Gabriela's actual password
- `YOUR_SABRINA_PASSWORD_HERE` → Sabrina's actual password
- `YOUR_MANAGER_PASSWORD_HERE` → the manager's actual password

#### Step 5: Copy the output
The console will print 4 lines like:
```
ADM: pbkdf2:a1b2c3d4e5f6...:7f8e9d0c1b2a...
Gabriela Ferreira: pbkdf2:f6e5d4c3b2a1...:0c1b2a3d4e5f...
...
```

#### Step 6: Update your `.env` file
Open `.env` and replace the old hashes:

```env
# BEFORE (old SHA-256 — 64 hex chars, no prefix):
VITE_ADM_HASH=5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8

# AFTER (new PBKDF2 — starts with "pbkdf2:"):
VITE_ADM_HASH=pbkdf2:a1b2c3d4e5f6...:7f8e9d0c1b2a...
VITE_HASH_GABRIELA=pbkdf2:f6e5d4c3b2a1...:0c1b2a3d4e5f...
VITE_HASH_SABRINA=pbkdf2:...
VITE_MANAGER_HASH=pbkdf2:...
```

#### Step 7: Restart the dev server
```bash
# Stop the current server (Ctrl+C), then:
npm run dev
```

### TASK B: Set Custom Claims via Firebase Admin SDK

**Why:** The new Firestore rules check for `request.auth.token.admin` and `request.auth.token.manager`. These are **custom claims** that must be set server-side using the Firebase Admin SDK. They cannot be set from the client.

#### Important: Your Auth Architecture

Your app uses **anonymous Firebase auth** (`signInAnonymously`) with a custom password layer on top. This means:

- Firebase doesn't know about "Gabriela" or "Sabrina" as separate users
- All users share one anonymous Firebase auth session
- Custom claims are tied to the Firebase Auth UID, not your app-level username

**Recommended Approach: Set admin claim on the anonymous user**

Since all your users share one Firebase anonymous auth session, setting `admin: true` on that single user will make the Firestore rules work correctly.

##### Step 1: Get the anonymous user UID
1. Open your app in the browser and log in
2. Open DevTools Console (`F12`)
3. Run: `firebase.auth().currentUser.uid`
4. Copy the UID (looks like: `aBcDeFgHiJkLmNoPqRsTuVwXyZ12`)

##### Step 2: Create the claims script
Create `scripts/set-claims.js`:

```javascript
const admin = require('firebase-admin');
const serviceAccount = require('./service-account-key.json');

admin.initializeApp({
  credential: admin.credential.cert(serviceAccount),
});

const auth = admin.auth();

async function setClaims() {
  // Replace with the actual anonymous user UID from Step 1
  const uid = 'PASTE_ANONYMOUS_USER_UID_HERE';

  try {
    await auth.setCustomUserClaims(uid, { admin: true });
    console.log('✅ Admin claim set successfully');
    console.log('⚠️ User must log out and back in for claims to take effect');
  } catch (err) {
    console.error('❌ Error:', err.message);
  }

  process.exit(0);
}

setClaims();
```

##### Step 3: Get your Service Account Key
1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project
3. Click the **gear icon** ⚙️ → **Project settings**
4. Go to the **Service accounts** tab
5. Click **Generate new private key**
6. Save the downloaded JSON file as `scripts/service-account-key.json`

##### Step 4: Install and run
```bash
npm install firebase-admin
node scripts/set-claims.js
```

##### Step 5: Re-login
Log out of the app and log back in. The `request.auth.token.admin` will now be `true` and all Firestore rules will work correctly.

## Deployment Plan

### Files to Commit to Git

**Safe to commit:**
- `firestore.rules` — Updated security rules with per-user isolation
- `package.json` / `package-lock.json` — Added firebase-admin dependency
- `scripts/generate-hashes.cjs` — PBKDF2 hash generator utility
- `scripts/set-claims.cjs` — Firebase custom claims setter utility
- `.gitignore` — Updated to protect sensitive files

**DO NOT commit (security-sensitive):**
- `scripts/service-account-key.json` — Contains private Firebase Admin SDK keys
- `.env` — Contains password hashes and API keys

### Pre-Commit Steps

1. Update `.gitignore` to block sensitive files:
   ```
   echo "scripts/service-account-key.json" >> .gitignore
   echo "scripts/*.cjs" >> .gitignore
   ```

2. Stage only safe files:
   ```bash
   git add firestore.rules package.json package-lock.json .gitignore
   ```

3. Commit with descriptive message:
   ```bash
   git commit -m "security: update Firestore rules and add hash/claims scripts"
   ```

4. Push to GitHub:
   ```bash
   git push origin main
   ```

### Firebase Deployment

1. Build production bundle:
   ```bash
   npm run build
   ```

2. Deploy to Firebase Hosting:
   ```bash
   firebase deploy --project miplace-premium-sales-control
   ```

   This deploys:
   - `dist/` to Firebase Hosting
   - `firestore.rules` to Cloud Firestore (already deployed, but safe to re-deploy)

### Post-Deployment Verification

1. Open the deployed app URL from Firebase Console
2. Log in with admin credentials
3. Verify all sales are visible
4. Test delete functionality
5. Check that `system_logs` writes succeed
